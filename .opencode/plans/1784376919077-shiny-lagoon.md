# Fix: Dynamic tenant CORS for GoTrue auth

## Root cause (two layers)

1. **Immediate:** GoTrue container crashed (empty `GOTRUE_JWT_SECRET`) because the shell that ran `docker compose up` didn't export `SUPABASE_JWT_SECRET`. The JWT secret exists in `backend/.env` but isn't auto-propagated.

2. **Architectural:** `GOTRUE_CORS_ALLOWED_ORIGINS` is a static comma-separated list -- when a new tenant signs up with a new subdomain (e.g., `newbiz.localhost:3000` or `newbiz.wren.app`), GoTrue rejects it because the origin isn't in the list.

The backend already solves this correctly via `app/main.py`:

```python
_ALLOWED_ORIGIN_REGEX = r"^https?://([a-z0-9-]+\.)?(localhost|wren\.app)(:\d+)?$"
```

The fix applies the same regex pattern in nginx so the auth-proxy handles CORS dynamically for ALL origins matching the tenant pattern.

## Changes

### 1. `docker/auth-proxy.conf` -- Rewrite CORS handling

Move all CORS logic from GoTrue to nginx. Use `map` + `$http_origin` to dynamically validate and reflect the origin, matching the backend's regex.

**Replace the entire `location /auth/v1/` block** with:

```nginx
location /auth/v1/ {
    set $gotrue http://auth:9999;

    # CORS: validate origin against the same regex the backend uses
    # (app/main.py _ALLOWED_ORIGIN_REGEX).  Reflect the valid origin
    # back so any tenant subdomain works immediately -- no static list.
    # If $http_origin is absent (curl, server-to-server), $cors_origin
    # stays empty and no Access-Control-Allow-Origin header is sent.
    set $cors_origin "";
    if ($http_origin ~* "^https?://([a-z0-9-]+\.)?(localhost|wren\.app)(:\d+)?$") {
        set $cors_origin $http_origin;
    }

    # Preflight: nginx returns immediately with CORS headers.
    # (GoTrue's internal CORS middleware cannot see the browser's
    # OPTIONS preflight through the proxy.)
    if ($request_method = 'OPTIONS') {
        add_header 'Access-Control-Allow-Origin' '$cors_origin' always;
        add_header 'Access-Control-Allow-Methods' 'GET, POST, PUT, PATCH, DELETE, OPTIONS' always;
        add_header 'Access-Control-Allow-Headers' 'content-type, apikey, authorization, x-client-info, x-supabase-api-version' always;
        add_header 'Access-Control-Max-Age' 86400 always;
        add_header 'Content-Length' 0;
        add_header 'Content-Type' 'text/plain';
        return 204;
    }

    # Strip any CORS header GoTrue emits (avoid duplicates) and add
    # the single correct one on every response -- including 502/504
    # errors from a crashed upstream -- so the browser never sees a
    # confusing "CORS error" for what is really a backend failure.
    proxy_hide_header Access-Control-Allow-Origin;
    proxy_hide_header Access-Control-Allow-Methods;
    proxy_hide_header Access-Control-Allow-Headers;
    add_header 'Access-Control-Allow-Origin' '$cors_origin' always;

    rewrite ^/auth/v1/(.*)$ /$1 break;
    proxy_pass $gotrue;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_pass_request_headers on;
}
```

### 2. `docker-compose.yml` -- Auth container healthcheck

Add a healthcheck to the `auth` service so Docker knows when GoTrue is actually ready (not just started), and make `auth-proxy` wait for it:

```yaml
auth:
  # ... existing config unchanged ...
  healthcheck:
    test: ["CMD", "wget", "-q", "--spider", "http://localhost:9999/health"]
    interval: 5s
    timeout: 3s
    retries: 10

auth-proxy:
  depends_on:
    auth:
      condition: service_healthy   # was: - auth
```

### 3. Runtime fix -- Restart auth with JWT secret

```bash
export SUPABASE_JWT_SECRET=9a02adf976b5e9cb9eeb721ebe2ad1e956b2ac892c999455e6b78829a51b0a59
docker compose up -d auth auth-proxy
```

## Why nginx handles CORS instead of GoTrue

| Layer | Before | After |
|-------|--------|-------|
| GoTrue | `GOTRUE_CORS_ALLOWED_ORIGINS` = static list of 4 origins | No CORS config needed (nginx strips its headers) |
| nginx | Only handles OPTIONS preflight with `*` | Handles ALL CORS: preflight + proxy responses + error responses |
| New tenant | Must be added to static list + GoTrue restart | Works immediately, zero config |

The `$http_origin` reflection is safe: the `if` regex only mirrors origins that match the tenant subdomain pattern -- `*.localhost`, `*.wren.app`, or bare `localhost` / `wren.app`. A malicious origin from any other domain is rejected (empty `$cors_origin`, so the browser blocks the request per the CORS spec).
