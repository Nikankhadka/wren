# Plan: E2E Auth Testing with Playwright

## Problem

1. Login at `http://app.localhost:3000/login` throws `TypeError: Failed to fetch` at `page.tsx:41` when calling `getSupabase().auth.signInWithPassword()` - the browser can't reach the GoTrue auth service.
2. The project has zero E2E test infrastructure - no Playwright, no Cypress, no browser tests.
3. No automated way to validate that demo credentials and auth infrastructure are correct.

## Root Cause Diagnosis (to be confirmed during build)

The "Failed to fetch" is a network-level failure (not an auth error). Likely causes, in priority order:

1. **Docker containers not running** - `docker compose up -d db auth auth-proxy` needs the GoTrue and auth-proxy services on port 54321. If these are down, the browser can't connect.
2. **Invalid/expired anon key** - If `SUPABASE_JWT_SECRET` in `backend/.env` changed after the anon key was minted (e.g., `scripts/demo.sh` was re-run), the `NEXT_PUBLIC_SUPABASE_ANON_KEY` in `frontend/.env.local` no longer matches what GoTrue expects.
3. **CORS** - GoTrue is configured with `GOTRUE_CORS_ALLOWED_ORIGINS` for `*.localhost:3000`, but if the browser sends unexpected headers, GoTrue will reject the preflight.

## Implementation Plan

### Step 1: Diagnose and Fix the Auth Error

- Run `docker ps` to check if `auth` and `auth-proxy` containers are running
- If not, run `docker compose up -d db auth auth-proxy`
- Verify GoTrue is reachable: `curl http://localhost:54321/auth/v1/health`
- Verify the anon key by re-minting it: `cd backend && uv run python -c "from seeds.supabase_keys import mint_anon_key; print(mint_anon_key())"` and compare with `frontend/.env.local`
- If they don't match, update `frontend/.env.local`
- Test login via GoTrue API directly: `curl -X POST http://localhost:54321/auth/v1/token?grant_type=password -H "apikey: <anon_key>" -d '{"email":"owner@bytefix.dev","password":"wren-demo"}'`

### Step 2: Install Playwright

Add to `frontend/package.json`:
- `@playwright/test` (devDependency)
- Add scripts: `test:e2e` (`playwright test`), `test:e2e:ui` (`playwright test --ui`)

Run `npm install` (or `npx playwright install` for browsers).

### Step 3: Create Playwright Config

**New file: `frontend/playwright.config.ts`**
- Test directory: `e2e/`
- Base URL: `http://app.localhost:3000` (tenant-admin surface)
- Web server config: reuse existing Next.js dev server (or start one if not running)
- Browser: chromium (headless for CI, headed for local dev)
- Timeout: 60s for auth flows
- Projects for dev (localhost) and CI (with retries)

### Step 4: Create E2E Test Helpers

**New file: `frontend/e2e/auth-helpers.ts`**
- `DEMO_USERS` constant with all 3 demo identities:
  - `owner@bytefix.dev` / `wren-demo` (tenant admin, Bytefix)
  - `owner@lumident.dev` / `wren-demo` (tenant admin, Lumident)
  - `founder@wren.dev` / `wren-demo` (platform admin)
- Helper functions: `loginAsTenantAdmin(page, email)`, `loginAsPlatformAdmin(page, email)`, `verifyAuthed(page)`

### Step 5: Write E2E Tests

**New file: `frontend/e2e/auth-login.spec.ts`**
Tests for tenant-admin login (`http://app.localhost:3000/login`):
- **Test 1**: Login with Bytefix owner - fills email/password, submits, verifies "Signed in" with tenant name "Bytefix Repairs"
- **Test 2**: Login with Lumident owner - fills email/password, submits, verifies "Signed in" with tenant name "Lumident Dental"
- **Test 3**: Invalid credentials - fills wrong password, verifies error message appears
- **Test 4**: Empty form submission - verifies HTML5 validation

**New file: `frontend/e2e/auth-platform.spec.ts`**
Tests for platform admin login (`http://admin.localhost:3000/login`):
- **Test 5**: Login with founder - fills email/password, submits, verifies redirect to `/` (tenants page)
- **Test 6**: Non-admin gets 403 - uses Bytefix owner to login on platform surface, verifies "not a platform admin" error

**New file: `frontend/e2e/auth-credentials-validation.spec.ts`**
Credential validation smoke test (first thing to run before other tests):
- **Test 0**: Direct GoTrue API health check - `fetch(http://localhost:54321/auth/v1/health)`
- **Test 0b**: Verify env vars exist - `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` are non-empty
- **Test 0c**: Verify each demo user can get a token from GoTrue directly (API-level, no browser)
- **Test 0d**: Verify backend health check - `fetch(http://localhost:8000/health)`

### Step 6: Add npm Scripts

Update `frontend/package.json`:
```json
"test:e2e": "playwright test",
"test:e2e:ui": "playwright test --ui",
"test:e2e:smoke": "playwright test --grep 'auth-credentials-validation'"
```

### Step 7: Verify

- Run `docker compose up -d db auth auth-proxy` (if not running)
- Run `cd backend && uv run python -m seeds.seed_demo` (seed demo data)
- Run `cd frontend && npm run test:e2e` to run all E2E tests
- Run `npm run test` to confirm existing vitest tests still pass
- Run `npm run lint` and `npm run typecheck` to confirm no regressions
